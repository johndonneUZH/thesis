\chapter{Background and related work}

\section{Vector Search}\label{sec:vector-search}

\subsection{Vector representation}

The representation of complex items (such as text, images, or user behavior) as high-dimensional vectors has become increasingly important in modern data-intensive applications. These vectors, commonly referred to as embeddings, are typically learnt using machine learning models and capture semantic or structural relationships in the data \cite{almeida2023wordembeddingssurvey}. As a result, semantically similar items are mapped to nearby points in a continuous vector space, enabling similarity to be measured using geometric distance functions such as cosine similarity or Euclidean distance.

The widespread adoption of this paradigm has led to the emergence of vector databases, which extend traditional database systems by enabling efficient similarity-based retrieval. Unlike conventional databases that operate on structured attributes using exact or range-based predicates, vector databases are designed to efficiently retrieve similar items from large collections of vectors by leveraging specialized index structures (e.g., graph-based or tree-based indexes) and approximate nearest neighbor search algorithms \cite{ma2026comprehensivesurveyvectordatabase}.

\subsection{Nearest Neighbor Search}

Given a dataset of vectors $\{x_1, \dots, x_n\} \subset \mathbb{R}^d$, the nearest neighbor search problem consists of identifying, for a query vector $q \in \mathbb{R}^d$, the vectors in the dataset that are most similar to $q$. This problem is also known as the \textit{post office problem}, where one seeks the closest facility to a given location. In the $k$-nearest neighbor ($k$NN) setting, the goal is to retrieve the $k$ most similar vectors. 

To formalize this notion of similarity, we use a function $D(\cdot, \cdot)$ that assigns a numerical score to a pair of vectors, reflecting how far apart they are in the underlying space. Common choices include Euclidean distance, cosine similarity, and inner product-based measures, each capturing different notions of similarity and directly influencing retrieval quality.

Given such a function, a natural solution is to compute $D(q, x_i)$ for every vector $x_i$ in the dataset and return the closest ones. This approach guarantees exact results, but requires scanning the entire dataset and performing a computation of cost $\mathcal{O}(d)$ per vector, resulting in a total time complexity of $\mathcal{O}(n \cdot d)$. As both the number of vectors $n$ and the dimensionality $d$ grow, this linear scan becomes computationally expensive and leads to high query latency \cite{arya1998optimal}. Moreover, in high-dimensional spaces, the contrast between nearby and far-away points tends to diminish, making it harder to distinguish meaningful neighbors \cite{indyk1998approximate}.

These limitations motivate the use of approximate nearest neighbor approaches.

\subsection{Approximate Nearest Neighbor Search}

Approximate nearest neighbor (ANN) methods trade exactness for efficiency by exploring only a subset of the dataset while aiming to preserve high recall. Instead of exhaustively comparing a query against all data points, ANN algorithms rely on specialized index structures and heuristics to efficiently identify promising candidates.

A prominent class is based on hashing techniques, such as locality-sensitive hashing (LSH), which project high-dimensional vectors into lower-dimensional hash spaces in a way that preserves similarity \cite{jafari2021surveylocalitysensitivehashing, gionis1999similarity}. The key idea behind LSH is to use hash functions that increase the probability of collision for similar vectors while reducing it for dissimilar ones. This enables efficient retrieval of candidate neighbors from hash buckets, often achieving sub-linear query time. The downside is that LSH-based methods may require multiple hash tables and can suffer from reduced accuracy or increased memory usage in high-dimensional settings.

Graph-based indexing methods have demonstrated particularly strong empirical performance. In particular, the Hierarchical Navigable Small World (HNSW) graph arranges data points in a multi-layer structure and uses greedy search to quickly reach nearest neighbors \cite{malkov2018efficientrobustapproximatenearest}. Due to its strong balance between efficiency and accuracy, HNSW has become widely adopted in modern vector search systems, but it typically requires the entire index to reside in main memory. For larger, disk-resident datasets, approaches such as DiskANN instead rely on a single-layer graph (Vamana) that is better suited for disk-based access, avoiding the multi-layer traversal of HNSW and enabling efficient search at billion scale \cite{subramanya2019diskann}.

Recent work has also explored learning-based approaches, where models are used to guide how the search structure is built or traversed \cite{salemohamed2024discoveringdatastructuresnearest, chiu2019learningindexnearestneighbor}. Instead of relying only on manually designed heuristics, these methods learn from the data how to partition the vector space or which regions are more likely to contain relevant neighbors. This makes the search process more efficient and highlights that ANN performance depends not only on the search algorithm, but also on how the data is organized.

In this thesis, ANN search is implemented using the FAISS library \cite{douze2025faisslibrary}. It provides a unified framework that implements a variety of ANN techniques, including graph-based methods such as HNSW, as well as exact search for ground truth computation.

\subsection{Metrics for ANNS Evaluation}\label{subsec:metrics-for-ann}

ANN systems are typically evaluated along three main dimensions: retrieval quality, query efficiency, and resource usage, reflecting the trade-offs inherent in approximate search \cite{johnson2017billionscalesimilaritysearchgpus, douze2025faisslibrary}.

A primary measure of retrieval quality is \textit{recall@k}, which quantifies how many of the true nearest neighbors are retrieved by the approximate method. Let $R_k^*$ denote the set of the $k$ exact nearest neighbors and $R_k$ the set returned by the ANN method. Then:
\[
\text{recall@k} = \frac{|R_k \cap R_k^*|}{|R_k^*|}.
\]

More generally, $n$-recall@k measures how many of the true nearest neighbors are successfully retrieved by the approximate method. Concretely, it is the fraction of the top-$n$ ground truth nearest neighbors that appear among the top-$k$ approximate results, where $n \leq k$ \cite{douze2025faisslibrary}. A common choice is $n = k$, in which case the metric reflects how well the approximate result set matches the exact one.

The definition of recall@k varies across the literature: some works use $1$-recall@k, focusing only on the nearest neighbor, while others use $k$-recall@k, which evaluates the retrieved set as a whole. In this thesis, we use the latter.

Query efficiency is typically measured in terms of latency and throughput (queries per second, QPS). Latency captures the time required to answer a single query, while throughput reflects the number of queries that can be processed per unit time. Together, these metrics determine how well a system scales under different workloads.

Resource usage further influences these metrics, including memory consumption, index construction time, and, in disk-based settings, I/O cost during search \cite{douze2025faisslibrary}. For example, graph-based indexes require additional memory to store connectivity information, while compression techniques reduce memory usage at the cost of lower recall.

\section{Filtered Vector Search}\label{sec:filtered-vector-search}

Let $(X, d)$ be a metric space, where $X \subseteq \mathbb{R}^d$ is a finite set of vectors and $d(\cdot,\cdot)$ is a distance function (e.g., Euclidean or cosine distance). Each vector $x \in X$ is associated with structured attributes $a(x) \in \mathcal{A}$, where $\mathcal{A}$ denotes the attribute space.

A \emph{predicate} is a Boolean function
\[
p : \mathcal{A} \rightarrow \{0,1\},
\]
which defines a subset of vectors
\[
X_p := \{ x \in X \mid p(a(x)) = 1 \}.
\]

Given a query vector $q \in \mathbb{R}^d$ and an integer $k$, the filtered $k$-nearest neighbor problem is to retrieve the $k$ closest vectors to $q$ among those that satisfy the predicate:
\[
R_k(q, p) = \text{top-}k \text{ elements of } X_p \text{ with respect to } d(q,\cdot).
\]

\medskip
The defining characteristic of filtered vector search is that the query is evaluated over the subset $X_p$, rather than over the full dataset $X$. That is, the search domain changes from $(X, d)$ to $(X_p, d)$. This restriction does not only reduce the number of candidate points; it changes the distribution and structure of the data on which the search is performed.

Formally, if $\mu$ denotes the empirical distribution over $X$, then restricting to $X_p$ corresponds to conditioning on the predicate:
\[
\mu_p(x) = \mathbb{P}\big(x \mid p(a(x)) = 1\big).
\]
In general, $\mu_p$ can differ significantly from $\mu$. For instance, $X_p$ may be much smaller, unevenly distributed, or composed of points that are far apart in the embedding space.

In the full dataset $X$, the points closest to $q$ may not satisfy the predicate and be excluded from $X_p$. Thus, the nearest neighbors of $q$ within $X_p$ can be entirely different from the nearest neighbors in $X$.

Each predicate $p$ defines therefore a different search problem. The subset $X_p$ can vary in size, distribution, and connectivity, and these variations determine how difficult the search is. As a result, filtered vector search is inherently \emph{query-dependent}: instead of operating on a single fixed dataset, the system must handle many different subsets defined at query time.

This creates a fundamental challenge. Most NN methods are designed under assumptions about the global structure of $X$. When applied to a subset $X_p$ that does not follow the same structure, these methods may become inefficient or lose accuracy. \cite{acorn, labelsvectors, nhq, caminal2025filtered}.

\subsection{Selectivity}\label{sec:selectivity}

Selectivity is a fundamental concept in query processing that quantifies the fraction of a dataset satisfying a predicate. Given a dataset \(X\) and a predicate \(p\), the selectivity of \(p\) is defined as
\[
\text{sel}(p) = \frac{|X_p|}{|X|}.
\]
The cardinality of \(p\) is given by \(|X_p|\).

Cardinality estimation has been extensively studied in the database literature due to its central role in query optimization. Classical approaches often assume attribute independence, approximating the selectivity of a conjunction of predicates as the product of their individual selectivities. While computationally convenient, this assumption can lead to large estimation errors in the presence of correlated attributes \cite{taskar2001}. Such estimates, combined with statistics such as histograms or samples, are used by query planners to select execution strategies, and inaccuracies can lead to suboptimal plans \cite{lynch2001selectivity, lpbound}.

In filtered vector search, selectivity determines the size of the candidate set. For high selectivity, \(|X_p|\) remains large and the search space closely resembles the full dataset. For low selectivity, \(|X_p|\) is small, reducing the number of candidates but restricting the available neighbors.

However, selectivity alone does not fully characterize query difficulty. Two predicates \(p_1, p_2\) with \(\text{sel}(p_1) = \text{sel}(p_2)\) may induce subsets \(X_{p_1}\) and \(X_{p_2}\) with very different structures in the embedding space, leading to different retrieval behavior. In this thesis, we therefore study queries across varying selectivity levels while explicitly accounting for attribute--embedding correlation.

\subsection{Correlation}\label{sec:correlation}

In addition to the size of the admissible set \(X_p\), performance also depends on how the predicate-induced subset is arranged in the embedding space relative to the query.

To formalize the interaction between the predicate and the embedding space, define
\[
g_k(q,S) = \frac{1}{k}\sum_{x \in \operatorname{NN}_k(q,S)} d(q,x),
\]
where \(\operatorname{NN}_k(q,S)\) denotes the \(k\) nearest neighbors of \(q\) in a set \(S\), and \(d\) is the distance function used for retrieval. Thus, \(g_k(q,X_p)\) is the average distance from \(q\) to its \(k\) nearest admissible neighbors under predicate \(p\) \cite{acorn}.

To determine whether these admissible neighbors are unusually close or unusually far, we compare them to a baseline in which the predicate has no geometric structure in the embedding space. Let
\[
R_p \subseteq X
\]
be a random subset of size \(|X_p|\), sampled uniformly from \(X\). Then \(R_p\) represents a hypothetical subset with the same cardinality as \(X_p\), but without any predicate-specific spatial organization.

We define the query-level correlation of \((q,p)\) as
\[
C(q,p) = \mathbb{E}[g_k(q,R_p)] - g_k(q,X_p),
\]
where the expectation is taken over the random draw of \(R_p\).

This quantity admits a natural interpretation:
\[
C(q,p) > 0 \quad \Longrightarrow \quad g_k(q,X_p) < \mathbb{E}[g_k(q,R_p)],
\]
meaning that the admissible neighbors in \(X_p\) are, on average, closer to the query than would be expected under a random subset of the same size. In this case, the query exhibits \emph{positive correlation}.

Conversely,
\[
C(q,p) < 0 \quad \Longrightarrow \quad g_k(q,X_p) > \mathbb{E}[g_k(q,R_p)],
\]
meaning that the admissible neighbors are farther from the query than expected under the random baseline. In this case, the query exhibits \emph{negative correlation}.

Finally,
\[
C(q,p) \approx 0
\]
indicates \emph{no correlation}, i.e., the predicate-induced subset behaves, from the perspective of the query, similarly to a random subset of the same size.

Intuitively, positive correlation means that the predicate selects vectors that are geometrically aligned with the query region: the valid answers tend to lie inside or near the query's local neighborhood. Negative correlation means the opposite: although vectors satisfying the predicate exist, they are systematically displaced from the query's neighborhood, so retrieving them requires exploration beyond the region that would be most promising under vector similarity alone.

This distinction is fundamental for filtered vector search. Two predicates may have the same selectivity, i.e.,
\[
|X_{p_1}| = |X_{p_2}|,
\]
while inducing very different values of \(C(q,p)\). In that case, the two queries have the same number of admissible vectors but differ substantially in geometric difficulty. Correlation therefore captures an aspect of filtered vector search that selectivity alone cannot express.

In this thesis, we develop a practical method to estimate this quantity efficiently.

\section{Execution Strategies}\label{sec:execution-strategies}

Execution strategies define how structured filter predicates are combined with nearest neighbor search. The main difference between them is when and how the filter is applied in relation to the search process. This design choice directly affects performance, efficiency, and result quality.

In this thesis, we focus on three main execution strategies, which will be later described in detail in Section~\ref{subsec:implementation-details}.

\subsection{Pre-filtering}\label{sec:pre-filtering}

Let \( D \) be a dataset of vectors and \( \phi \) a predicate. Pre-filtering first evaluates the predicate and constructs the subset
\[
D_\phi = \{ x \in D \mid \phi(x) \}.
\]
Nearest neighbor search is then performed only over \( D_\phi \).

This guarantees that all returned results satisfy the predicate. However, standard vector indexes are built over the full dataset \( D \) and cannot be directly restricted to arbitrary subsets \( D_\phi \). In practice, this often means scanning all vectors in \( D_\phi \) and computing distances explicitly, with cost proportional to \( |D_\phi| \). When selectivity is high (\( |D_\phi| \approx |D| \)), the cost is close to a full scan of the dataset.

\subsection{Post-filtering}\label{sec:post-filtering}

Post-filtering performs nearest neighbor search over \( D \), ignoring the predicate. Let \( C_k \subseteq D \) denote the top-\(k\) retrieved candidates. The predicate is then applied to \( C_k \), and only elements satisfying \( \phi \) are kept.

This allows the index to operate normally, but it does not guarantee that \(k\) valid results are obtained. If only a fraction \( \text{sel}(\phi) \) of the dataset satisfies the predicate, the expected number of valid results in \( C_k \) is approximately \( k \cdot \text{sel}(\phi) \). If this number is less than \(k\), additional candidates must be retrieved, increasing the total number of distance evaluations.

\subsection{Inline Filtering}\label{sec:inline-filtering}

Inline filtering incorporates the predicate directly into the search, restricting traversal to nodes that satisfy the constraint. 
In graph-based indexes, this corresponds to searching over the subgraph induced by the predicate, which may be poorly connected and thus difficult to explore effectively.

ACORN~\cite{acorn} addresses this issue by preserving connectivity under filtering. 
It first performs standard HNSW routing until reaching a node satisfying the predicate, and then continues the search within the filtered subgraph by applying the predicate during neighbor expansion.

We use ACORN as the representative inline filtering approach.
