\chapter{Background and related work}

\section{Vector Search}\label{sec:vector-search}

\subsection{Vector representation}

Modern data-intensive applications increasingly rely on representing complex objects (e.g., text, images, or user behavior) as high-dimensional vectors. These vector representations, often referred to as embeddings, are typically learned using machine learning models and capture semantic or structural relationships in the data \cite{almeida2023wordembeddingssurvey}. As a result, semantically similar objects are mapped to nearby points in a continuous vector space, allowing similarity between objects to be measured using geometric distance functions such as Euclidean distance or cosine similarity.

The widespread adoption of embeddings has led to the emergence of vector databases, which extend traditional database systems by supporting efficient similarity-based retrieval over high-dimensional vectors. In contrast to conventional databases that primarily operate on structured attributes and exact-match queries, vector databases are designed to handle nearest neighbor search over embeddings at scale, enabling applications such as semantic search, recommendation systems, and retrieval-augmented generation. \cite{ma2026comprehensivesurveyvectordatabase}

\subsection{Nearest Neighbor Search}

Given a dataset of vectors $\{x_1, \dots, x_n\} \subset \mathbb{R}^d$, the nearest neighbor search problem consists of identifying the vectors that are closest to a query vector $q \in \mathbb{R}^d$ according to a distance (or dissimilarity) measure. In the $k$-nearest neighbor ($k$NN) setting, the goal is to retrieve the $k$ closest vectors to $q$. This problem is also commonly referred to as the \textit{post office problem}, where the objective is to find the closest facility to a given location.

The notion of “closeness” is defined through a distance function $D(\cdot, \cdot)$, which quantifies dissimilarity between vectors. Common choices include the Euclidean distance, cosine distance, and inner product-based measures. The choice of distance function plays a critical role, as it determines how similarity in the embedding space is interpreted and directly affects retrieval quality.

A straightforward solution is to compute the distance between the query and every vector in the dataset. While this exact approach guarantees optimal results, it has a time complexity of $\mathcal{O}(n \cdot d)$, which becomes prohibitive for large-scale datasets and high-dimensional spaces \cite{arya1998optimal}.

Nearest neighbor search is a fundamental problem in many domains, including information retrieval, machine learning, and computer vision \cite{indyk1998approximate}. However, as the dimensionality of the data increases, exact search methods suffer from the so-called \textit{curse of dimensionality}, where distance computations become increasingly expensive and less discriminative.

These limitations motivate the use of approximate methods, which trade exactness for efficiency and enable scalable similarity search in high-dimensional spaces.

\subsection{Approximate Nearest Neighbor Search}

To address the scalability limitations of exact nearest neighbor search, approximate nearest neighbor (ANN) methods have been developed. These methods trade exactness for efficiency by exploring only a subset of the dataset while aiming to preserve high recall. Instead of exhaustively comparing a query against all data points, ANN algorithms rely on specialized index structures and heuristics to efficiently identify promising candidates.

ANN methods can be broadly categorized into different families. A prominent class is based on hashing techniques, such as locality-sensitive hashing (LSH), which project high-dimensional vectors into lower-dimensional hash spaces in a way that preserves similarity \cite{indyk1998approximate, jafari2021surveylocalitysensitivehashing, gionis1999similarity}. The key idea behind LSH is to use hash functions that increase the probability of collision for similar vectors while reducing it for dissimilar ones. This allows candidate neighbors to be retrieved efficiently from hash buckets, enabling sub-linear query time. However, LSH-based methods may require multiple hash tables and can suffer from reduced accuracy or increased memory usage in very high-dimensional settings.

Graph-based indexing methods have demonstrated particularly strong empirical performance. In particular, the Hierarchical Navigable Small World (HNSW) graph organizes data points into a multi-layer structure that enables efficient navigation toward nearest neighbors through greedy search \cite{malkov2018efficientrobustapproximatenearest}. Due to its balance between efficiency and accuracy, HNSW has become a widely adopted method in modern vector search systems. To further improve scalability to very large datasets, including those stored on disk, systems such as DiskANN extend graph-based approaches by optimizing data layout and access patterns for disk-based storage \cite{subramanya2019diskann}. These methods enable efficient nearest neighbor search over billion-scale datasets while maintaining high accuracy.

In addition to classical ANN structures, recent work has explored how data structures for nearest neighbor search can themselves be discovered or learned from data \cite{salemohamed2024discoveringdatastructuresnearest}. Similarly, learning-based indexing approaches have been proposed in which a model is trained to predict or guide the partitioning of the vector space so that search can be performed more efficiently \cite{chiu2019learningindexnearestneighbor}. These approaches illustrate that ANN performance depends not only on the search algorithm, but also on how the underlying search structure is constructed.

In this thesis, ANN search is implemented using the FAISS library, a widely used system for efficient similarity search over high-dimensional vectors \cite{douze2025faisslibrary}. FAISS provides a unified framework that implements a variety of ANN techniques, including graph-based methods such as HNSW, as well as exact search for ground truth computation. Its scalability and performance make it a standard choice for large-scale vector search applications.

\subsection{Metrics for ANNS Evaluation}

Approximate nearest neighbor methods trade exactness for efficiency, making it necessary to evaluate both retrieval quality and computational performance. As a result, ANN systems are typically assessed along two main dimensions: retrieval accuracy and resource usage \cite{johnson2017billionscalesimilaritysearchgpus, douze2025faisslibrary}.

A primary measure of retrieval quality is \textit{recall@k}, which quantifies how many of the true nearest neighbors are retrieved by the approximate method. Let $R_k^*$ denote the set of the $k$ exact nearest neighbors and $R_k$ the set returned by the ANN method. Then:
\[
\text{recall@k} = \frac{|R_k \cap R_k^*|}{|R_k^*|}.
\]

More generally, $n$-recall@k is defined as the fraction of the top-$n$ ground truth nearest neighbors that appear in the top-$k$ approximate results, where $n \leq k$ \cite{douze2025faisslibrary}. A common setting is $n = k$, in which case the measure corresponds to the intersection between the exact and approximate result sets. When $n = k = 1$, recall@1 reduces to a binary indicator of whether the nearest neighbor was correctly retrieved, and is therefore equivalent to accuracy. For $k > 1$, however, recall@k measures the fraction of correctly retrieved neighbors and should not be confused with accuracy in the general classification sense.

It is worth noting that the definition of recall@k is not entirely consistent across the literature. In some works, recall@k refers to $1\text{-recall@k}$, while in others it corresponds to $k$-recall@k. In this thesis, we adopt the standard definition based on the fraction of correctly retrieved nearest neighbors.

In addition to retrieval accuracy, computational efficiency is a key metric. Query performance is typically measured in terms of latency (time per query) or throughput, often expressed as queries per second (QPS). These metrics capture the practical efficiency of ANN systems and are central to evaluating their scalability.

ANN methods also introduce additional resource considerations. These include memory usage for storing index structures, preprocessing or index construction time, and, in large-scale or disk-based systems, the number of input/output operations required during search \cite{douze2025faisslibrary}. For example, graph-based indexes require additional memory to store connectivity information, while compressed representations may reduce memory at the cost of reduced retrieval accuracy.

Overall, ANN evaluation involves analyzing trade-offs between retrieval accuracy and resource usage. Algorithms are typically tuned to achieve an optimal balance, where high recall is maintained while minimizing query time and resource consumption.

In this thesis, ANN methods are evaluated using recall@k to measure retrieval accuracy, and queries per second (QPS) and query latency to assess performance. Exact nearest neighbor search is used to compute ground truth results for comparison.

\section{Filtered Vector Search}\label{sec:filtered-vector-search}

Filtered vector search extends nearest neighbor search by combining similarity-based retrieval in an embedding space with structured filtering over associated attributes. Given a query vector and a predicate over metadata, the goal is to retrieve the nearest neighbors that satisfy the predicate. This setting arises naturally in modern applications, where retrieval must simultaneously account for semantic similarity and structured constraints.

The difficulty arises because the filter does not simply reduce the number of candidates; it changes the effective search domain. For a predicate \(p\), the query is evaluated over the subset \(X_p \subseteq X\) of vectors whose records satisfy \(p\), rather than over the full dataset. The geometry of \(X_p\) in the embedding space can differ significantly from that of \(X\): it may be dense and well connected, or sparse and fragmented. Consequently, vectors satisfying the predicate need not lie near the query, even when they are the only admissible answers \cite{acorn, labelsvectors}.

This makes filtered vector search inherently query-dependent. Each predicate induces a different subset \(X_p\), and therefore a different effective search space. Unlike standard nearest neighbor search, where a single index is built over a fixed distribution, filtered queries operate over many implicit subspaces defined at query time. These subspaces vary in size, density, and connectivity, which directly affects search behavior.

As a result, methods designed under assumptions about the global data distribution may behave inconsistently when applied to predicate-induced subsets. Existing literature reports inefficiencies, increased computational cost, and degraded retrieval quality when the filtered subset is poorly aligned with the structure of the embedding space \cite{nhq, caminal2025filtered}.

\subsection{Selectivity}\label{sec:selectivity}

Selectivity is a fundamental concept in query processing that quantifies the fraction of the dataset satisfying a given predicate. Let \(X\) denote the dataset and \(p\) a predicate. The subset of elements satisfying \(p\) is defined as
\[
X_p = \{ x \in X \mid p(x) = \text{true} \}.
\]
The selectivity of \(p\) is then given by
\[
\text{sel}(p) = \frac{|X_p|}{|X|}.
\]
Selectivity is closely related to cardinality, which refers to the absolute number of matching elements, i.e., \(|X_p|\).

Cardinality estimation has been extensively studied in the database literature, as it plays a central role in query optimization \cite{lynch2001selectivity, lpbound}. Classical approaches often rely on simplifying assumptions such as attribute independence, where the selectivity of a conjunction of predicates is approximated as the product of their individual selectivities. While this assumption simplifies estimation, it can lead to significant errors when attributes are correlated, as it ignores dependencies between them \cite{taskar2001, lpbound}. Query planners rely on these estimates, together with other statistics such as histograms or sampling-based summaries, to choose efficient execution strategies. Inaccurate estimates can propagate through the planning process, leading to suboptimal decisions.

In filtered vector search, selectivity directly determines the size of the admissible search space. Given a query vector \(q\), retrieval is performed over the restricted domain \(X_p\) rather than the full dataset \(X\). When \(\text{sel}(p)\) is high, the search space remains large and resembles the original dataset. When \(\text{sel}(p)\) is low, the search is restricted to a small subset, which can significantly reduce the number of candidate vectors but also increase the difficulty of finding relevant neighbors. In particular, if \(|X_p|\) is small, the nearest valid neighbors may lie far from the query in the embedding space, requiring more extensive exploration.

However, selectivity alone does not fully characterize query difficulty. Two predicates \(p_1\) and \(p_2\) with \(\text{sel}(p_1) = \text{sel}(p_2)\) may induce subsets \(X_{p_1}\) and \(X_{p_2}\) with very different structures in the embedding space, leading to different retrieval behavior. In this thesis, we therefore evaluate queries across different selectivity levels while focusing on the effect of attribute–embedding correlation.

\subsection{Correlation}\label{sec:correlation}

As discussed above, selectivity alone does not fully characterize the difficulty of filtered vector search. In addition to the size of the admissible set \(X_p\), performance also depends on how the predicate-induced subset is arranged in the embedding space relative to the query.

Let
\[
X = \{x_1,\dots,x_n\} \subset \mathbb{R}^d
\]
be the set of embeddings, let \(p\) be a predicate, and let
\[
X_p = \{x \in X \mid p(x)=\text{true}\}
\]
denote the subset of vectors satisfying \(p\). For a query vector \(q \in \mathbb{R}^d\), filtered vector search seeks the \(k\) nearest neighbors of \(q\) within \(X_p\).

To formalize the interaction between the predicate and the embedding space, define
\[
g_k(q,S) = \frac{1}{k}\sum_{x \in \operatorname{NN}_k(q,S)} d(q,x),
\]
where \(\operatorname{NN}_k(q,S)\) denotes the \(k\) nearest neighbors of \(q\) in a set \(S\), and \(d\) is the distance function used for retrieval. Thus, \(g_k(q,X_p)\) is the average distance from \(q\) to its \(k\) nearest admissible neighbors under predicate \(p\) \cite{acorn}.

To determine whether these admissible neighbors are unusually close or unusually far, we compare them to a baseline in which the predicate has no geometric structure in the embedding space. Let
\[
R_p \subseteq X
\]
be a random subset of size \(|X_p|\), sampled uniformly from \(X\). Then \(R_p\) represents a hypothetical subset with the same cardinality as \(X_p\), but without any predicate-specific spatial organization.

We define the query-level correlation of \((q,p)\) as
\[
C(q,p) = \mathbb{E}[g_k(q,R_p)] - g_k(q,X_p),
\]
where the expectation is taken over the random draw of \(R_p\).

This quantity admits a natural interpretation:
\[
C(q,p) > 0 \quad \Longrightarrow \quad g_k(q,X_p) < \mathbb{E}[g_k(q,R_p)],
\]
meaning that the admissible neighbors in \(X_p\) are, on average, closer to the query than would be expected under a random subset of the same size. In this case, the query exhibits \emph{positive correlation}.

Conversely,
\[
C(q,p) < 0 \quad \Longrightarrow \quad g_k(q,X_p) > \mathbb{E}[g_k(q,R_p)],
\]
meaning that the admissible neighbors are farther from the query than expected under the random baseline. In this case, the query exhibits \emph{negative correlation}.

Finally,
\[
C(q,p) \approx 0
\]
indicates \emph{no correlation}, i.e., the predicate-induced subset behaves, from the perspective of the query, similarly to a random subset of the same size.

Intuitively, positive correlation means that the predicate selects vectors that are geometrically aligned with the query region: the valid answers tend to lie inside or near the query's local neighborhood. Negative correlation means the opposite: although vectors satisfying the predicate exist, they are systematically displaced from the query's neighborhood, so retrieving them requires exploration beyond the region that would be most promising under vector similarity alone.

This distinction is fundamental for filtered vector search. Two predicates may have the same selectivity, i.e.,
\[
|X_{p_1}| = |X_{p_2}|,
\]
while inducing very different values of \(C(q,p)\). In that case, the two queries have the same number of admissible vectors but differ substantially in geometric difficulty. Correlation therefore captures an aspect of filtered vector search that selectivity alone cannot express.

Correlation therefore captures a fundamental aspect of filtered vector search that selectivity alone cannot express. In this thesis, we develop a practical method to estimate this quantity efficiently.
