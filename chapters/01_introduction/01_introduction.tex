\chapter{Introduction}
\section{Motivation and Problem Setting}

Consider a user searching for a shirt that is visually similar to a high-end item, but costs less than a given threshold, for example under 30.
This query combines two requirements: a vector-similarity condition, which retrieves visually or stylistically related items, and a structured predicate, which restricts the result set by an attribute such as price.

A first factor that affects query cost is predicate selectivity.
If the predicate is highly selective, only a small fraction of the dataset remains admissible.
As a result, the search may need to inspect more candidates before it finds enough points that both satisfy the predicate and are close to the query.

\begin{figure}[htbp]
    \centering
    \includegraphics[width=0.8\textwidth]{figures/motivation_selectivity.png}
    \caption{Effect of predicate selectivity on filtered vector search.}
    \label{fig:motivation-selectivity}
\end{figure}

However, selectivity alone does not determine query difficulty.
Two predicates with the same selectivity can still lead to different search costs, 
because the points satisfying the predicate may be concentrated in different parts of the embedding space.
If the filter is aligned with vector locality, many valid points lie near the query and the filtered set overlaps with the neighborhood that the vector index would naturally explore.
The search can therefore recover valid nearest neighbors with little additional work.
If the filter is not aligned with the embedding space, the nearby neighborhood may contain mostly points that fail the predicate.
The search must then continue farther through the index before finding enough valid results.

    \caption{Effect of attribute--embedding correlation.}
    \label{fig:motivation-correlation}
\end{figure}

The central problem studied in this thesis is how to quantify this relationship between structured attributes and vector embeddings.

\section{Research Gap}\label{sec:research-gap}

Existing work on filtered vector search often evaluates systems using synthetic or semi-synthetic structured attributes.
This is not a limitation by itself: synthetic workloads are useful because they make it possible to control selectivity, predicate type, and query complexity.
The limitation is that, in many common benchmarks, these workloads do not control the relationship between attributes and vector geometry.

A common starting point for this type of benchmark construction is Milvus~\cite{milvus}, which is later referenced by several filtered vector search papers.
Milvus takes vectors from SIFT1B~\cite{sift1b} and augments each vector with a randomly generated scalar attribute.
Filtered predicates are then created over this attribute, making it possible to vary selectivity in a controlled way.
However, because the attribute is sampled randomly, the benchmark does not control where the satisfying vectors lie in the embedding space.

This design choice is partly caused by the lack of public datasets that naturally combine embeddings with structured attributes.
AnalyticDB-V~\cite{analyticdbv} states this issue explicitly, noting that ``so far, there is no publicly available dataset that contains both structured data and unstructured data.''
Accordingly, it uses public vector-only datasets such as SIFT1B~\cite{sift1b} and Deep1B~\cite{deep1b} for standard ANN experiments, but relies on the in-house AliCommodity dataset for hybrid-query experiments.
Since this dataset is not publicly available, later work must either use alternative real-world datasets or construct semi-synthetic hybrid benvchmarks.

Milvus~\cite{milvus}, Multiattribute ANNS~\cite{multiattributenhq}, and NHQ~\cite{nhq} follow the latter approach.

Filtered-DiskANN~\cite{filtereddiskann} explicitly notes the limitation of decoupling vectors and attributes.
The authors argue that such datasets are less realistic because ``in real-world datasets there could be correlation between the distribution of points and the set of labels that an ANNS algorithm could exploit.''

This thesis addresses this gap by explicitly modeling attribute--embedding correlation and studying how this signal can support filtered vector search planning.

\section{Main Goals}

This thesis has three main goals.

First, we propose a family of correlation metrics for filtered vector search.
These metrics measure how much the local neighborhood of a query differs from the global dataset with respect to a structured attribute distribution.
Their purpose is to capture whether a filter is aligned with the region of the embedding space that the vector index is likely to explore.
During the evaluation, we study three concrete variants of this idea and identify the most effective one.

Second, we evaluate these metrics empirically.
We analyze whether they can distinguish between positive, negative, and random query--filter relationships,
and whether these distinctions explain differences in query difficulty and execution behavior.

Third, we study how the selected metric can be integrated into a filtered vector search system.
In particular, we investigate how it can be used for query planning and how it may complement existing state-of-the-art techniques such as ACORN.
