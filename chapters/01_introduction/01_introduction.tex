\chapter{Introduction}
\section{Motivation and Problem Setting}

Consider a user searching for a shirt that is visually similar to a high-end item, but costs less than a given threshold, for example under 30.
This query combines two requirements: a vector-similarity condition, which retrieves visually or stylistically related items, and a structured predicate, which restricts the result set by an attribute such as price.

Queries of this form are common in modern retrieval systems, where unstructured similarity search is combined with structured filtering.
The system must retrieve items that are close to the query in the embedding space, while also satisfying the user-specified filter.
The challenge is therefore not only to find similar items, but to find similar items inside a restricted subset of the data.

A first factor that affects query cost is predicate selectivity.
If the predicate is highly selective, only a small fraction of the dataset remains admissible.
As a result, the search may need to inspect more candidates before it finds enough points that both satisfy the predicate and are close to the query.
Lower selectivity therefore often increases retrieval cost.

However, selectivity alone does not fully determine query difficulty.
Two predicates with the same selectivity can behave very differently depending on where their valid points lie in the embedding space.
If the filtered points are concentrated near the query, the search can recover valid nearest neighbors with little additional work.
If they are mostly far from the query, the search must explore a larger and less relevant region of the index.

\begin{figure}[htbp]
    \centering
    \includegraphics[width=0.8\textwidth]{figures/motivation_correlation.png}
    \caption{Effect of attribute--embedding correlation.}
    \label{fig:motivation-correlation}
\end{figure}

This second effect depends on the relationship between structured attributes and the embedding space.
When the filter attribute is aligned with vector locality, filtering preserves the neighborhood around the query.
In this case, the admissible set contains many points that the vector index would naturally explore.
When the attribute is not aligned with the embedding space, filtering disrupts this neighborhood structure.
The search may then encounter many nearby points that fail the predicate and must continue farther through the index.

The central problem studied in this thesis is how to quantify this relationship between structured attributes and vector embeddings.
Such a measure can help explain why filtered vector search queries with similar selectivity may have different costs, and it can support query-planning decisions between execution strategies such as pre-filtering, post-filtering, and hybrid graph-based search.

\section{Research Gap}\label{sec:research-gap}

Existing work, however, typically handles this interaction only indirectly and in a system-specific manner. In particular, most evaluations rely on synthetic workloads where attributes are generated independently of the vector data.

Starting from NHQ \cite{nhq}, prior work constructs such workloads by attaching synthetic attributes to vectors, following approaches used in systems such as Milvus \cite{milvus} and Multiattribute ANNS \cite{multiattributenhq}. In these settings, attribute values are assigned by randomly sampling from predefined categories. While this enables controlled experiments, it ignores any connection between attributes and embeddings. Similarly, Filtered-DiskANN \cite{filtereddiskann} generates labels independently of the vector space, explicitly removing any correlation.

As a result, current approaches do not capture how filters and vectors interact in practice, and this effect remains largely unmodeled.
