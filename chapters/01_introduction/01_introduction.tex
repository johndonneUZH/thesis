\chapter{Introduction}
\section{Motivation and Problem Setting}

Consider a user searching for a shirt similar to a high-end item, but with a price below a given threshold (e.g., under 30). This query combines two components: similarity, to find visually or stylistically related items, and filtering, to enforce a constraint on structured attributes such as price.

This type of query is representative of modern applications, where vector search is combined with structured predicates. In such settings, the goal is not only to retrieve similar items, but to do so within a restricted subset of the data.

Processing these queries efficiently is challenging. While vector search techniques exploit the geometric structure of the embedding space, filtering restricts the search to a subset of the data. This restriction not only reduces the number of candidate points, but can also change their distribution in the embedding space.

A key factor is how the filtered vectors are positioned relative to the query. In some cases, the filter selects vectors that are concentrated near the query, making them easy to retrieve. In other cases, they are located farther away, requiring the search to explore less relevant regions of the space.

This difference reflects how structured attributes interact with the embedding space. When attributes and embeddings are aligned, filtering preserves the local neighborhood of the query. When they are not aligned, filtering disrupts this structure and makes the search more difficult.

Capturing this effect is essential for efficient query processing, as it directly influences how different execution strategies behave and how much work is required to retrieve relevant results.

\section{Research Gap}\label{sec:research-gap}

Existing work, however, typically handles this interaction only indirectly and in a system-specific manner. In particular, most evaluations rely on synthetic workloads where attributes are generated independently of the vector data.

Starting from NHQ \cite{nhq}, prior work constructs such workloads by attaching synthetic attributes to vectors, following approaches used in systems such as Milvus \cite{milvus} and Multiattribute ANNS \cite{multiattributenhq}. In these settings, attribute values are assigned by randomly sampling from predefined categories. While this enables controlled experiments, it ignores any connection between attributes and embeddings. Similarly, Filtered-DiskANN \cite{filtereddiskann} generates labels independently of the vector space, explicitly removing any correlation.

As a result, current approaches do not capture how filters and vectors interact in practice, and this effect remains largely unmodeled.

\section{Research Objectives \& Questions}

The main objective is to treat this relationship as a property of the data and to define a quantitative measure that captures how filter predicates relate to the vector distribution. This enables a more systematic understanding of how filtering affects nearest neighbor search.

This leads to the following research questions:

\begin{itemize}
    \item How can the relationship between filter predicates and the vector space be formally defined and measured?
    \item How does this relationship affect the behavior and performance of hybrid search methods?
    \item How can this information be used to improve query processing beyond assumptions of independence?
\end{itemize}

The goal is to develop a general and system-independent framework that can be applied across different index structures and query execution methods, including both exact and approximate search.

\section{Thesis Structure}

The remainder of this thesis is organized as follows.

Chapter 2 provides the necessary background and reviews related work. It introduces vector search, approximate nearest neighbor methods, and filtered vector search, including key concepts such as selectivity, correlation, and common execution strategies.

Chapter 3 formalizes the problem setting by defining the data model, query model, and the objective of the study.

Chapter 4 describes the construction of the datasets used in this thesis. In particular, it explains how both correlated and uncorrelated attributes are generated, as well as how query ranges and selectivity are controlled.

Chapter 5 introduces the proposed correlation metric. It presents the comparison between local and global attribute distributions, the use of Jensen–Shannon divergence, and the incorporation of lift to capture query-dependent effects.

Chapter 6 describes the system setup and the execution strategies considered, including pre-filtering, post-filtering, and inline filtering, along with implementation details.

Chapter 7 outlines the experimental setup, including evaluation metrics and the computation of ground truth.
